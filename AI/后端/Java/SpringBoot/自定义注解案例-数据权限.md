# 自定义注解实战：SpringBoot 数据权限注解完整拆解

## 一、先通俗讲透：注解到底是干嘛的？

### 1. 底层本质

注解只是**贴在类 / 方法上的标签**，本身不会执行任何逻辑；

真正干活的是 **AOP 切面**，切面通过反射拿到这个标签，读取标签上配置的参数，再统一实现通用逻辑。

### 2. 业务痛点（为什么需要数据权限注解）

后台系统常见需求：**不同角色只能看到自己的数据**

举例：

- 超级管理员：查看全部订单
- 门店管理员：只能查看本店订单
- 普通员工：只能查看自己创建的订单

如果不用注解，每个查询订单接口都要写一堆判断、拼接 SQL 过滤条件，大量重复代码，难以维护。

解决方案：自定义注解 `@DataScope`，贴在接口方法上，AOP 统一拦截，自动拼接数据权限 SQL，**一行注解搞定权限控制**。

### 3. 项目文件分层（标准 SpringBoot 业务结构）

```
src/main/java/com/demo/
├── annotation        # 所有自定义注解存放包
│   └── DataScope.java  # 自定义数据权限注解

```

# 二、完整分步实现

## 步骤 1：自定义注解 `@DataScope`

作用：作为标签，标记接口需要数据权限过滤，同时携带配置参数（表别名、部门字段、用户字段）

```
package com.demo.annotation;

import java.lang.annotation.*;

/**
 * 自定义数据权限注解
 * 贴在Controller查询方法上，开启数据自动过滤
 */
@Target(ElementType.METHOD) // 仅允许标注在方法上
@Retention(RetentionPolicy.RUNTIME) // 运行时保留，AOP反射读取
@Documented
public @interface DataScope {

    // sql表别名，默认t
    String tableAlias() default "t";

    // 部门字段名：门店id/部门id
    String deptColumn() default "dept_id";

    // 创建人字段：创建用户id
    String userColumn() default "create_user";
}
```

### 注解参数说明

1. `tableAlias`：mybatis sql 的表别名，比如 `select * from order t`，t 就是别名
2. `deptColumn`：数据库部门字段，门店管理员根据此字段过滤
3. `userColumn`：创建人 ID 字段，普通员工根据此字段过滤

## 步骤 2：用户上下文工具类（模拟登录用户信息）

存放当前登录用户的角色、所属部门、用户 ID，切面需要读取：

```
package com.demo.util;

/**
 * 登录用户上下文，存储当前登录人信息
 */
public class UserContext {
    // 模拟登录用户实体
    public static class LoginUser {
        // 角色 1超级管理员 2门店管理员 3普通员工
        private Integer roleType;
        private Long deptId; // 当前用户所属门店/部门id
        private Long userId; // 当前用户id

        // getter setter
        public Integer getRoleType() {return roleType;}
        public Long getDeptId() {return deptId;}
        public Long getUserId() {return userId;}
    }

    // 存储当前登录用户（ThreadLocal保证线程隔离）
    private static final ThreadLocal<LoginUser> USER_THREAD_LOCAL = new ThreadLocal<>();

    public static void setLoginUser(LoginUser user) {
        USER_THREAD_LOCAL.set(user);
    }

    public static LoginUser getLoginUser() {
        return USER_THREAD_LOCAL.get();
    }

    public static void clear() {
        USER_THREAD_LOCAL.remove();
    }
}
```

## 步骤 3：AOP 切面类（注解真正执行逻辑）

核心：拦截所有加了 `@DataScope` 的接口方法，读取注解配置，拼接 SQL 过滤条件存入 ThreadLocal，Mybatis 拦截器自动拼接 SQL。

```
package com.demo.aspect;

import com.demo.annotation.DataScope;
import com.demo.util.UserContext;
import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.aspectj.lang.annotation.Pointcut;
import org.aspectj.lang.reflect.MethodSignature;
import org.springframework.stereotype.Component;
import java.lang.reflect.Method;

@Aspect
@Component
public class DataScopeAspect {

    // 定义切点：所有带@DataScope注解的方法
    @Pointcut("@annotation(com.demo.annotation.DataScope)")
    public void dataScopePointCut() {}

    // 方法执行前拦截
    @Before("dataScopePointCut()")
    public void handleDataScope(JoinPoint point) {
        // 1. 获取当前方法上的@DataScope注解
        MethodSignature signature = (MethodSignature) point.getSignature();
        Method method = signature.getMethod();
        DataScope dataScope = method.getAnnotation(DataScope.class);

        // 2. 读取注解上配置的参数
        String tableAlias = dataScope.tableAlias();
        String deptCol = dataScope.deptColumn();
        String userCol = dataScope.userColumn();

        // 3. 获取当前登录用户
        UserContext.LoginUser loginUser = UserContext.getLoginUser();
        Integer role = loginUser.getRoleType();
        Long deptId = loginUser.getDeptId();
        Long userId = loginUser.getUserId();

        // 4. 根据角色拼接数据权限过滤SQL片段
        String filterSql = "";
        if (role == 1) {
            // 超级管理员：无过滤，查看全部
            filterSql = "";
        } else if (role == 2) {
            // 门店管理员：只看本店数据
            filterSql = tableAlias + "." + deptCol + " = " + deptId;
        } else if (role == 3) {
            // 普通员工：只看自己创建的数据
            filterSql = tableAlias + "." + userCol + " = " + userId;
        }

        // 5. 将拼接好的sql存入线程上下文，Mybatis拦截器读取拼接
        UserContext.setDataFilterSql(filterSql);
    }
}
```

## 步骤 4：Mybatis 拦截器（读取切面生成的 SQL，自动拼接查询语句）

```
package com.demo.interceptor;

import com.demo.util.UserContext;
import org.apache.ibatis.executor.statement.StatementHandler;
import org.apache.ibatis.plugin.Interceptor;
import org.apache.ibatis.plugin.Intercepts;
import org.apache.ibatis.plugin.Signature;
import java.sql.Connection;

@Intercepts({
        @Signature(type = StatementHandler.class, method = "prepare", args = {Connection.class, Integer.class})
})
public class DataScopeInterceptor implements Interceptor {
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        StatementHandler handler = (StatementHandler) invocation.getTarget();
        String originSql = handler.getBoundSql().getSql();
        String filterSql = UserContext.getDataFilterSql();

        // 存在数据权限条件，拼接where过滤
        if (filterSql != null && !filterSql.isEmpty()) {
            String newSql = originSql + " AND " + filterSql;
            // 替换原始sql
            ReflectUtil.setFieldValue(handler.getBoundSql(), "sql", newSql);
        }
        return invocation.proceed();
    }
}
```

## 步骤 5：Controller 接口使用注解（业务层极简代码）

只需要在查询方法上加 `@DataScope`，不用写任何权限判断代码

```
package com.demo.controller;

import com.demo.annotation.DataScope;
import com.demo.service.OrderService;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;
import javax.annotation.Resource;
import java.util.List;

@RestController
@RequestMapping("/order")
public class OrderController {

    @Resource
    private OrderService orderService;

    /**
     * 查询订单列表，自动数据权限过滤
     * 注解不传参，使用默认表别名t、默认字段
     */
    @GetMapping("/list")
    @DataScope
    public List<Order> listOrder() {
        return orderService.listAll();
    }

    /**
     * 自定义注解参数：自定义表别名o、自定义部门字段shop_id
     */
    @GetMapping("/page")
    @DataScope(tableAlias = "o", deptColumn = "shop_id")
    public Page<Order> pageOrder() {
        return orderService.page();
    }
}
```

## 步骤 6：Mapper 原始 SQL（完全不用手动加权限条件）

```
<select id="listAll" resultType="Order">
    select id, order_no, dept_id, create_user from order t
</select>
```

执行时，AOP + Mybatis 拦截器自动拼接过滤条件，最终执行 SQL：

门店管理员登录：

```
select id, order_no, dept_id, create_user from order t AND t.dept_id = 1001
```

普通员工登录：

```
select id, order_no, dept_id, create_user from order t AND t.create_user = 2002
```

# 三、完整业务执行流程（一条请求走完全链路）

1. 前端请求 `/order/list` 接口；
2. 请求进入 `listOrder()` 方法，方法上标注了 `@DataScope`；
3. AOP 切面 `DataScopeAspect` 触发拦截，切点匹配成功；
4. 切面通过反射读取方法上 `@DataScope` 的配置参数；
5. 获取当前登录用户角色，自动拼接对应的数据过滤 SQL；
6. 把过滤 SQL 存入 `ThreadLocal` 上下文；
7. Mapper 执行查询 SQL 时，Mybatis 拦截器读取上下文的过滤条件，自动拼接 SQL；
8. 返回过滤后的数据，实现不同角色数据隔离。

# 四、回答核心疑问：注解在这里到底起了什么作用？

1. **标记作用**：区分哪些接口需要数据权限过滤。不加 `@DataScope` 的接口，切面不会拦截，不会拼接过滤 SQL；
2. **传参载体**：注解内部定义 `tableAlias`、`deptColumn`，给切面传递自定义配置，适配不同业务表；
3. **统一解耦**：权限逻辑全部抽离到 AOP，Controller 业务代码干净，不用重复写 if 判断、拼接 SQL；
4. **可插拔**：哪个接口需要数据权限，只需要贴一行注解，不需要的直接删掉注解即可。

# 五、对比无注解的原始写法（凸显注解优势）

## 不用注解，每个接口都要写重复逻辑

```
@GetMapping("/list")
public List<Order> listOrder() {
    // 每个接口重复写一遍角色判断、拼接sql
    UserContext.LoginUser user = UserContext.getLoginUser();
    String sqlFilter = "";
    if(user.getRoleType() == 2){
        sqlFilter = "t.dept_id = " + user.getDeptId();
    }else if(user.getRoleType() ==3){
        sqlFilter = "t.create_user = " + user.getUserId();
    }
    return orderService.list(sqlFilter);
}
```

缺点：

- 大量重复代码，每个查询接口复制粘贴；
- 逻辑分散，修改权限规则需要改所有接口；
- 业务代码混杂权限逻辑，可读性差。

## 使用注解后

Controller 无任何权限相关代码，全部统一由 AOP 处理，统一维护、统一修改。

# 六、拓展：日常开发同类注解场景（举一反三）

1. **@Log 操作日志注解**：贴接口，AOP 自动记录操作人、操作内容、耗时；
2. **@RateLimit 限流注解**：贴接口，限制接口每秒访问次数；
3. **@Encrypt 字段加密注解**：实体类字段标记，序列化自动加密；
4. **@RepeatSubmit 防重复提交注解**：拦截短时间重复请求。

所有这类通用横切逻辑，全部使用「自定义注解 + AOP」实现，注解只做标记传参，切面实现核心逻辑。
