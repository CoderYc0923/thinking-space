# Axios/Fetch调用AI API：大模型流式响应（SSE/WebSocket）处理总结

# 一、核心概念

前端通过Axios/Fetch调用AI API时，大模型流式响应是指模型不等待完整结果生成后一次性返回，而是将结果按片段（Token级/句子级）实时推送至前端的传输方式。核心价值是**降低用户等待焦虑、提升交互流畅性**，尤其适用于长文本生成、多轮对话、代码生成等耗时场景。

主流流式传输协议及适配场景：

|传输协议|核心特性|Axios/Fetch适配性|典型应用场景|
|---|---|---|---|
|SSE（Server-Sent Events）|单向通信（服务端→前端）、基于HTTP协议、轻量无状态、默认支持重连|均支持（需配置响应类型为流/文本）|单轮长文本生成（如文档总结、文案创作）、实时消息推送|
|WebSocket|双向通信（服务端↔前端）、基于TCP协议、持久连接、低延迟|不直接适配（需单独创建WebSocket实例，Axios/Fetch可用于鉴权预处理）|多轮对话、实时协作生成、需要前端主动中断/调整的生成场景|
Axios/Fetch核心角色：作为前端发起请求、接收流式数据的核心工具，负责请求封装、身份认证、流式数据解析与分发，是连接前端交互与AI API的关键链路。

# 二、核心技术实现：Axios/Fetch处理流式响应

## 2.1 基础前提：AI API流式响应规范

主流AI API（如OpenAI、文心一言）流式响应遵循统一规范：

- SSE：响应格式为 `data: {JSON数据}\n\n`，结束标记为 `data: [DONE]`；

- WebSocket：连接建立后，服务端通过二进制/文本帧推送JSON格式片段，需约定结束帧标识。

前端核心任务：按协议规范解析流数据、实时渲染、处理异常与中断。

## 2.2 Axios处理SSE流式响应

Axios支持通过 `responseType: 'stream'` 接收流式数据，需配合Node.js风格的流事件（data/end/error）处理。

### 2.2.1 核心代码实现（OpenAI API示例）

```javascript

import axios from 'axios';

/**
 * Axios调用OpenAI API（SSE流式响应）
 * @param {string} prompt 用户提示词
 * @param {Function} onChunk 每段流数据的回调（用于实时渲染）
 * @param {Function} onComplete 流结束回调
 * @param {Function} onError 错误回调
 * @returns {Promise<any>} 流实例（用于中断）
 */
const axiosStreamSSE = async (prompt, onChunk, onComplete, onError) => {
  try {
    const response = await axios.post(
      'https://api.openai.com/v1/chat/completions',
      {
        model: 'gpt-3.5-turbo',
        messages: [{ role: 'user', content: prompt }],
        stream: true, // 开启流式
        temperature: 0.7
      },
      {
        responseType: 'stream', // 关键配置：响应类型为流
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${process.env.VITE_OPENAI_API_KEY}` // 从环境变量获取密钥
        }
      }
    );

    const stream = response.data;
    let fullContent = '';

    // 监听流数据事件
    stream.on('data', (chunk) => {
      // 解析SSE格式：将Buffer转为字符串，按\n\n分割
      const chunkStr = chunk.toString('utf-8');
      const lines = chunkStr.split('\n\n').filter(line => line.trim());

      lines.forEach(line => {
        const dataStr = line.replace(/^data: /, ''); // 去除前缀
        if (dataStr === '[DONE]') return; // 结束标记，跳过
        
        try {
          const data = JSON.parse(dataStr);
          const chunkContent = data.choices[0]?.delta?.content || '';
          if (chunkContent) {
            fullContent += chunkContent;
            onChunk(fullContent); // 推送当前完整片段给前端渲染
          }
        } catch (err) {
          onError(`流数据解析失败：${err.message}`);
        }
      });
    });

    // 监听流结束事件
    stream.on('end', () => {
      onComplete(fullContent);
    });

    // 监听流错误事件
    stream.on('error', (err) => {
      onError(`流传输错误：${err.message}`);
    });

    return stream; // 返回流实例，用于后续中断
  } catch (err) {
    onError(`请求失败：${err.response?.data?.error?.message || err.message}`);
  }
};

// 调用示例
const streamInstance = axiosStreamSSE(
  '总结前端流式响应处理核心要点',
  (content) => console.log('实时渲染：', content), // 逐段渲染
  (fullContent) => console.log('完整结果：', fullContent),
  (err) => console.error('错误：', err)
);

// 如需中断流（如用户点击"停止生成"）
// streamInstance.destroy();

```

### 2.2.2 关键注意事项

- 浏览器环境兼容：Axios的stream事件依赖Node.js流API，浏览器环境需确保Axios版本适配（推荐v1.0+）；

- 中断控制：通过返回的stream实例调用 `destroy()` 可中断流传输，避免无效数据继续推送；

- 错误捕获：需同时捕获请求级错误（如401/404）和流传输级错误（如数据解析失败）。

## 2.3 Fetch处理SSE流式响应

Fetch API是浏览器原生API，通过 `ReadableStream` 处理流式数据，无需依赖第三方库，更适合纯浏览器环境。

### 2.3.1 核心代码实现（文心一言API示例）

```javascript

/**
 * Fetch调用文心一言API（SSE流式响应）
 * @param {string} prompt 用户提示词
 * @param {Function} onChunk 实时渲染回调
 * @param {Function} onComplete 结束回调
 * @param {Function} onError 错误回调
 * @returns {AbortController} 中断控制器（用于中断请求）
 */
const fetchStreamSSE = async (prompt, onChunk, onComplete, onError) => {
  const abortController = new AbortController();
  const signal = abortController.signal;

  try {
    const response = await fetch(
      'https://aip.baidubce.com/rpc/2.0/ai_custom/v1/wenxinworkshop/chat/eb-instant',
      {
        method: 'POST',
        signal: signal, // 关联中断控制器
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${process.env.VITE_BAIDU_API_KEY}`
        },
        body: JSON.stringify({
          messages: [{ role: 'user', content: prompt }],
          stream: true,
          temperature: 0.7
        })
      }
    );

    if (!response.ok) {
      const errData = await response.json();
      throw new Error(errData.error_msg || `HTTP错误：${response.status}`);
    }

    // 获取ReadableStream并创建读取器
    const reader = response.body.getReader();
    const decoder = new TextDecoder('utf-8'); // 用于将Uint8Array转为字符串
    let fullContent = '';
    let buffer = ''; // 缓存未完整解析的片段

    // 循环读取流数据
    const readStream = async () => {
      const { done, value } = await reader.read();

      if (done) {
        // 流结束，处理剩余缓存
        if (buffer) processChunk(buffer);
        onComplete(fullContent);
        return;
      }

      // 解码并拼接缓存
      buffer += decoder.decode(value, { stream: true });
      processChunk(buffer);

      // 继续读取下一段
      readStream();
    };

    // 解析SSE片段
    const processChunk = (chunk) => {
      const lines = chunk.split('\n\n');
      // 最后一段可能不完整，留到下次处理
      buffer = lines.pop() || '';

      lines.forEach(line => {
        if (!line.trim()) return;
        const dataStr = line.replace(/^data: /, '');
        if (dataStr === '[DONE]') return;

        try {
          const data = JSON.parse(dataStr);
          const chunkContent = data.result || ''; // 文心一言流式字段为result
          if (chunkContent) {
            fullContent += chunkContent;
            onChunk(fullContent);
          }
        } catch (err) {
          onError(`流解析失败：${err.message}`);
        }
      });
    };

    // 启动流读取
    readStream();

  } catch (err) {
    if (signal.aborted) {
      onError('请求已中断');
      return;
    }
    onError(`请求失败：${err.message}`);
  }

  return abortController; // 返回中断控制器
};

// 调用示例
const abortController = fetchStreamSSE(
  '解释Fetch与Axios处理流式响应的差异',
  (content) => console.log('实时渲染：', content),
  (fullContent) => console.log('完整结果：', fullContent),
  (err) => console.error('错误：', err)
);

// 中断请求（如用户取消）
// abortController.abort();

```

### 2.3.2 关键优势与注意事项

- 原生兼容：无需依赖第三方库，浏览器原生支持，适合轻量项目；

- 中断控制：通过 `AbortController` 实现请求中断，兼容性更好（支持所有现代浏览器）；

- 解码处理：需手动通过 `TextDecoder` 解码二进制流，需处理缓存拼接避免片段丢失。

## 2.4 WebSocket流式响应处理（Axios/Fetch预处理）

WebSocket适用于双向通信场景（如多轮对话需实时推送上下文），Axios/Fetch通常用于WebSocket连接前的鉴权（如获取临时Token），核心流式处理通过WebSocket原生API实现。

### 2.4.1 核心代码实现（含Axios鉴权）

```javascript

import axios from 'axios';

/**
 * WebSocket流式调用AI API（含Axios鉴权预处理）
 * @param {string} prompt 初始提示词
 * @param {Function} onChunk 实时渲染回调
 * @param {Function} onComplete 结束回调
 * @param {Function} onError 错误回调
 * @returns {WebSocket|null} WebSocket实例
 */
const websocketStream = async (prompt, onChunk, onComplete, onError) => {
  try {
    // 1. Axios预处理：获取WebSocket连接所需的临时Token
    const authRes = await axios.get('/api/ai/get-websocket-token', {
      params: { model: 'gpt-4' }
    });
    const { token, websocketUrl } = authRes.data;

    // 2. 创建WebSocket连接
    const ws = new WebSocket(`${websocketUrl}?token=${token}`);

    // 3. 连接成功：发送初始请求
    ws.onopen = () => {
      ws.send(JSON.stringify({
        messages: [{ role: 'user', content: prompt }],
        stream: true
      }));
    };

    // 4. 接收流式数据
    let fullContent = '';
    ws.onmessage = (event) => {
      const data = JSON.parse(event.data);
      if (data.type === 'chunk') { // 约定chunk类型为流式片段
        const chunkContent = data.content;
        fullContent += chunkContent;
        onChunk(fullContent);
      } else if (data.type === 'complete') { // 约定complete为结束
        onComplete(fullContent);
        ws.close(); // 结束后关闭连接
      }
    };

    // 5. 错误处理
    ws.onerror = (error) => {
      onError(`WebSocket错误：${error.message}`);
    };

    // 6. 连接关闭
    ws.onclose = (event) => {
      if (event.wasClean) {
        console.log('WebSocket连接正常关闭');
      } else {
        onError(`WebSocket意外关闭：${event.code} ${event.reason}`);
      }
    };

    return ws;
  } catch (err) {
    onError(`鉴权失败：${err.message}`);
    return null;
  }
};

// 调用示例
const wsInstance = await websocketStream(
  '进行多轮对话，逐步分析前端性能优化方案',
  (content) => console.log('实时渲染：', content),
  (fullContent) => console.log('完整结果：', fullContent),
  (err) => console.error('错误：', err)
);

// 发送后续对话（双向通信特性）
// wsInstance.send(JSON.stringify({
//   messages: [{ role: 'user', content: '再补充一下打包优化的具体方案' }]
// }));

// 关闭连接
// wsInstance.close();

```

### 2.4.2 适用场景与核心要点

- 双向通信需求：多轮对话、实时协作生成等需要前端主动推送后续指令的场景；

- 鉴权预处理：Axios/Fetch常用于获取WebSocket连接的临时凭证，避免直接暴露密钥；

- 连接管理：需处理重连机制（如网络中断后自动重连）、心跳检测（避免连接超时）。

# 三、核心场景案例

## 3.1 场景1：多轮对话机器人（WebSocket）

**场景描述**：用户与AI进行连续对话，AI需基于历史上下文实时响应，支持用户随时打断并发起新问题。

**技术选型**：WebSocket（双向通信）+ Axios（鉴权）

**核心实现要点**：

- 前端维护对话历史数组，每次发送请求时携带完整历史上下文；

- 实现“停止生成”按钮，点击时调用 `ws.close()` 中断当前流；

- 对话气泡实时渲染：每接收一段流数据，更新对应AI气泡的内容，模拟“打字机”效果。

## 3.2 场景2：代码生成工具（SSE + Fetch）

**场景描述**：用户输入需求（如“写一个React流式组件”），AI实时生成代码并高亮显示，支持复制完整代码。

**技术选型**：SSE（单向流，代码生成无需前端主动推送）+ Fetch（原生无依赖）

**核心实现要点**：

- 集成Highlight.js：每接收一段代码片段，调用高亮函数实时渲染（如 `hljs.highlightElement(codeElement)`）；

- 代码分段处理：按代码块（函数/类）边界优化渲染，避免片段断裂导致高亮异常；

- 加载状态提示：生成过程中显示“生成中...”动画，流结束后显示“复制成功”按钮。

## 3.3 场景3：长文档总结（SSE + Axios）

**场景描述**：用户上传万字文档，AI实时总结核心要点，分段推送结果（如“核心观点1/5”）。

**技术选型**：SSE（单向流，总结过程无需交互）+ Axios（支持大文件请求配置）

**核心实现要点**：

- 文档预处理：前端先读取文档内容（如PDF转文本），按Token阈值拆分后通过Axios流式提交；

- 进度可视化：基于接收的片段数/预估总片段数，显示总结进度条（如“3/5 核心观点生成中”）；

- 错误重试：流传输中断时，通过Axios重新发起请求，携带已生成的片段上下文，避免重复生成。

# 四、工程化优化要点

## 4.1 错误处理与容错

- 分类错误提示：区分“鉴权失败”“网络中断”“流解析失败”“API限流”等场景，给出针对性引导（如限流时提示“请1分钟后重试”）；

- 自动重试机制：网络波动导致的流中断，通过防抖函数延迟1-2秒自动重试，重试3次失败后提示用户；

- 数据兜底：流中断后保留已生成的片段，允许用户基于片段继续生成或手动保存。

## 4.2 性能优化

- 节流渲染：高频流片段推送时（如每10ms一段），通过 `requestAnimationFrame` 节流渲染，避免DOM频繁重绘；

- 内存管理：流结束后释放读取器/流实例，清理事件监听（如 `stream.off('data')`），避免内存泄漏；

- 预加载处理：高频使用的模型，提前通过Axios/Fetch获取鉴权Token或预热连接，减少首次生成延迟。

## 4.3 安全与合规

- 密钥保护：Axios/Fetch调用AI API时，密钥通过后端转发（前端不直接存储），避免密钥泄露；

- 输入过滤：前端过滤恶意提示词（如Prompt注入），避免调用AI生成不良内容；

- 跨域处理：SSE/WebSocket跨域时，需后端配置 `Access-Control-Allow-Origin` 等CORS头，WebSocket跨域需后端支持。

# 五、总结

Axios/Fetch调用AI API处理流式响应的核心是“协议适配+实时解析+流畅交互”：SSE适合单向长文本生成场景，Axios/Fetch均能实现，Fetch更具原生兼容性；WebSocket适合双向多轮对话场景，Axios/Fetch常用于预处理鉴权。实际落地需结合场景选择协议，重点优化流解析稳定性、渲染流畅性与错误容错能力。通过工程化优化（如节流渲染、自动重试、密钥保护），可进一步提升用户体验与系统稳定性，让大模型流式能力高效落地于前端应用。
